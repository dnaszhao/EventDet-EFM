# 强化特征建模的高效事件视觉目标检测方法：阶段性实验结果稿

## 1. 研究目标

本课题围绕事件视觉场景下的目标检测任务展开，目标是在有限标注和有限算力条件下，构建兼顾检测精度与运行效率的事件视觉目标检测方法。与传统帧式相机不同，事件相机输出的是异步事件流 `(x, y, t, p)`，具有高时间分辨率、低延迟和高动态范围等特点，但也带来了数据稀疏、噪声较多、表征方式特殊等问题。因此，如何在轻量化约束下强化事件特征建模能力，是本课题的核心研究问题。

结合当前工程实现与实验进展，本文阶段性工作重点并非提出全新的检测框架，而是在现有事件视觉检测模型基础上，完成本地训练环境搭建、弱监督训练流程打通、最优模型筛选以及定性结果可视化，为后续论文定量对比、误差分析和改进模块设计提供可靠基础。

## 2. 实验平台与数据集

### 2.1 实验平台

当前实验在本地单机环境下完成，主要软硬件条件如下：

| 项目 | 配置 |
| --- | --- |

| Python 环境 | Conda 环境 `myenv` |
| Python 版本 | 3.9.23 |
| PyTorch 版本 | 2.7.0 + CUDA 12.8 |
| 训练显卡 | NVIDIA GeForce RTX 3060 Laptop GPU |
| 显存容量 | 6 GB |

该平台已经能够稳定完成数据读取、模型训练、断点续训、checkpoint 保存、可视化推理等任务，说明本课题已具备完整实验推进条件。

### 2.2 数据集设置

本阶段实验采用 `Gen1` 事件视觉目标检测数据集。数据集包含 `car` 和 `pedestrian` 两类目标，适合用于验证事件流场景下的目标检测能力。项目中采用的事件表示形式为 `stacked_histogram_dt=50_nbins=10`，即将短时事件流编码为堆叠直方图形式，以兼顾事件时序信息与卷积/视觉主干的处理便利性。

本次训练实际使用的配置为 `dataset=gen1x0.01_ss`，对应 1% 标注比例的弱监督设定。该设置能够在较少标注下验证模型的基本检测能力，也更符合本课题关于“高效检测”和“有限标注利用”的研究背景。训练与可视化过程中使用的输入分辨率为 `240×304`，模型内部根据骨干网络需要自动 pad 到 `256×320`。

## 3. 模型与训练配置

本阶段采用的检测模型为 `RNNDet tiny` 配置，其主干为 `MaxViTRNN`，检测头为 `YOLOX`，属于参数规模较小、适合 6GB 显存单卡训练的轻量化方案。根据训练日志统计，模型总参数量约为 `4.4M`。

核心训练配置如下：

| 项目 | 设置 |
| --- | --- |
| 模型 | `rnndet` |
| Backbone | `MaxViTRNN-tiny` |
| FPN | `PAFPN` |
| Head | `YOLOX` |
| 训练精度 | AMP 16-bit |
| 学习率 | `2e-4` |
| batch size | `train=1`, `eval=1` |
| 序列长度 | `21` |
| 类别数 | `2` |
| 训练设备 | `hardware.gpus=0`，即独显 RTX 3060 |

实验过程中已经确认训练任务实际运行在 NVIDIA 独立显卡上，而非核显；训练日志中显示 `GPU available: True (cuda), used: True`，且设备名明确识别为 `NVIDIA GeForce RTX 3060 Laptop GPU`。

截至当前，本地训练已经能够稳定持续推进并自动保存最优与最新 checkpoint。现有模型权重中，最佳验证结果对应的 checkpoint 为：

```text
checkpoint/rnndet_tiny-gen1-bs1_iter400k/models/epoch_001-step_81756-val_AP_0.1270.ckpt
```

同时，项目中还保留了同等验证指标的另一份最优权重：

```text
checkpoint/rnndet_tiny-gen1-bs1_iter400k/models/epoch_001-step_80001-val_AP_0.1270.ckpt
```

继续训练得到的最新续训权重为：

```text
checkpoint/rnndet_tiny-gen1-bs1_iter400k/models/last_epoch_001-step_93383.ckpt
```

由此可见，在 `Gen1 + 1% 标注比例 + tiny 模型` 的设定下，当前模型已经在验证阶段达到 `val/AP = 0.1270` 的阶段性最优结果。虽然这一结果仍有进一步提升空间，但已经能够说明以下几点：

1. 当前事件数据读取、事件表示构建、检测模型前向传播、损失计算与参数更新流程均工作正常。
2. 在弱监督低标注场景下，模型已经具备基础目标检测能力，而不是完全无法收敛。
3. 当前配置可以作为后续改进实验、消融实验与论文对比实验的稳定基线。

需要说明的是，完整测试集的正式量化评估仍可在后续补充；由于本地单卡环境下 `test` 集评估耗时较长，本稿当前以最佳验证 checkpoint 的量化结果与测试集可视化结果作为阶段性实验依据。这种写法适合用于论文中期检查、阶段汇报或初稿实验部分。

## 8. 复现实验命令

本次阶段性结果对应的可视化命令如下：

```powershell
python vis_pred.py model=rnndet dataset=gen1 dataset.path=./datasets/gen1/ checkpoint="./checkpoint/rnndet_tiny-gen1-bs1_iter400k/models/epoch_001-step_81756-val_AP_0.1270.ckpt" +experiment/gen1=tiny_win3060.yaml model.postprocess.confidence_threshold=0.1 num_video=3 vis_sequence_length=160 reverse=False
```

为适配 6GB 显存，本次可视化将 `vis_sequence_length` 设置为 `160`。同时，对 `vis_pred.py` 做了两处工程化修正：一是避免在正常导出正向视频后错误退出；二是增加可覆盖的 `vis_sequence_length` 参数，以便在不同显存条件下稳定生成论文插图。
